# /write-stakeholder-update -- Executive Stakeholder Update

Draft the actual update artifact — document, message, or slide content — calibrated to the right altitude for your audience.

## Invocation

```
/stakeholder-update weekly PM update for my manager — here's what happened this sprint
/stakeholder-update monthly CPO update to CEO — missed GMV by 8%
/stakeholder-update board update for Thursday — Q1 review
```

## Workflow

Apply the **executive-stakeholder-update** skill to:

1. Detect level (IC/PM → manager, VP/CPO → CEO/leadership, or board-level) and update cadence
2. Gather the period's headline events, metrics, blockers, and decisions needed
3. Produce the full update in the right format and tone for the audience
4. Offer to refine for Slack, email, or slide format

## Notes

- Different from `/stakeholder-map` — that maps the landscape; this produces the actual communication
- MODE A (team): under 400 words, direct, scannable
- MODE B (leadership): 1 page max, data-anchored, decision-ready
- MODE C (board): 2 pages max, investor-grade narrative
